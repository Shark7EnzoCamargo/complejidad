Propuesta Objetivos Aplicacion

Propuesta

Planeamos utilizar la teoria de grafos para estructurar los datos extraidos de Google Maps.

Ello con el fin de hallar el camino minimo entre puntos de interes que el usuario desee transitar.

Ademas, haremos uso del algoritmo de Dijkstra para hallar el recorrido minimo de un grafo ponderado basandonos en el peso de las aristas.

El valor de las aristas seran determinadas teniendo en cuenta el nivel del trafico usual que puede existir en una calle o avenida y la distancia de los distritos principales de Lima Metropolitana.

Objetivos

  • Hallar la ruta minima entre distintos puntos de interes que el usuario haya seleccionado.
  • Permitir al usuario elegir el punto de partida y su destino
  • Mostrar la ruta detalladamente mediante un grafo.
  • Brindar la opción de seleccionar la hora al escoger una ruta.
  • Permitir que el usuario visualice los nombres de los lugares que recorra.
  • Incorporar un dataset completo de todos los lugares por cada distrito de Lima Metropolitana.
  • Implementar una interfaz simple y eficaz para que el usuario realice las búsquedas sin ningún problema.
  • Utilizar archivos .csv como base del dataset del programa.
  • Implementar algoritmos de recorridos de grafos.

    • Aplicacion

    El aplicativo consistiría de una ventana la cual contendrá 2 textbox donde el usuario podrá ingresar el lugar tanto de partida como el de llegada, como también sus botones respectivos para poder ingresar la elección del usuario.

    Además, se contará con un botón que generará una ventana con todos los lugares disponibles para su búsqueda categorizados por distritos.

    Posteriormente al ingreso de los lugares de partida y llegada, a través de un botón el aplicativo mostrará mediante un grafo la ruta más corta hacia el destino deseado.

    Finalmente, se incluirá un checkbox para que el usuario pueda definir si la búsqueda de la ruta será en la mañana, tarde o noche.
